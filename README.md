# CROWN SHIFT 👑

**参加者が本気で遊べて、見ているリスナーにも状況が伝わる**ことを最優先に設計した、配信向けオンライン同時選択バトルです。

- 2〜8人のプレイヤー
- 観客は別枠で参加
- 1試合 約8〜12分
- インストール不要、URLと6文字コードで参加
- 専用ゲームサーバー不要（ホストのブラウザが進行役）
- PC・スマートフォン対応

## 公開URL

GitHub Pagesを有効化すると、次のURLで遊べます。

**https://sujuuu81.github.io/-/**

初回のみ、リポジトリの **Settings → Pages → Build and deployment → Source** を **GitHub Actions** に設定してください。以後は `main` への更新時に自動公開されます。

## ゲームの狙い

毎ラウンド、全員が「移動先」と「作戦」を同時に伏せて選びます。公開時に全員の選択が一気に解決されます。中央の「王冠の間」と毎ラウンド変わる「注目エリア」でGLORYを稼ぎ、規定ラウンド終了時または勝利点到達時の首位が勝者です。

脱落はありません。体力が0になるとGLORYを失って即復帰するため、参加者が途中から暇になりません。

## 観客ができること

- 次ラウンドの世界ルール投票
- 次のラウンド首位予想（的中でHYPE +100）
- 会場リアクション（🔥 😂 😱 👑 👏）
- 最終結果で「観客王」も発表

観客投票は全体ルールだけを変え、特定のプレイヤーへ直接ダメージを与えません。人気投票だけで勝敗が決まらない設計です。

## 作戦

| 作戦 | 効果 |
|---|---|
| ⚔️ ストライク | 移動後、指定相手と同じエリアなら2ダメージ |
| 🛡️ ガード | 受けるダメージを合計2軽減。防御成功でGLORY +1 |
| 🫳 スナッチ | 移動後、指定相手と同じエリアならGLORYを1奪う |
| 💣 バースト | 移動後、同じエリアにいる他プレイヤー全員へ1ダメージ |
| 🎭 フェイント | 他の攻撃処理が終わってから移動。狙われた場所を抜ける |

## 配信での始め方

1. 配信者が「部屋を作る」
2. 画面のURLまたは6文字コードを共有
3. リスナーは「プレイヤー」か「観客」を選んで入室
4. 参加者が足りなければCPUを追加
5. 2人以上でゲーム開始

ホストがタブを閉じると部屋も終了します。配信中はゲームのタブを開いたままにしてください。

## 技術構成

- HTML / CSS / Vanilla JavaScript
- PeerJSによるWebRTCデータ通信
- ホスト権威型
- 選択フェーズ中、他プレイヤーの作戦内容はクライアントへ送信しない秘匿設計
- 切断したプレイヤーはCPUが自動で引き継ぎ
- GitHub ActionsによるGitHub Pages公開

## 通信上の注意

専用サーバーを持たないP2P方式なので無料で動きますが、企業・学校・一部モバイル回線などWebRTCが強く制限された環境では接続できない場合があります。安定運用の目安は、プレイヤー2〜8人＋観客数十人程度です。

## ローカル起動

```bash
python -m http.server 8000
```

その後 `http://localhost:8000` を開きます。
